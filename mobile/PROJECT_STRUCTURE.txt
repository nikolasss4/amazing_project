Key File Purposes
=================

Configuration Files:
- app.json: Expo settings (app name, icons, iOS/Android config)
- tsconfig.json: TypeScript strict mode + path aliases
- babel.config.js: Reanimated plugin for animations
- package.json: Dependencies, scripts, project metadata

App Entry:
- App.tsx: Renders RootNavigator + AssistantOverlay, sets up gesture handler

Navigation:
- RootNavigator.tsx: Bottom tab navigator with 3 screens
- CustomTabBar.tsx: Custom UI (left pill with 3 tabs, right pill with AI button)

Theme:
- tokens.ts: ALL design values (colors, spacing, typography, shadows, animations)
  * Never hardcode values, always use tokens
  * Enables consistent liquid glass aesthetic

State Management:
- store/index.ts: Zustand stores for Trade, Learn, and Assistant
  * TradeStore: selected pair, order type, amount
  * LearnStore: XP, streak, current scenario
  * AssistantStore: messages, screenshot URI, isOpen state

Features:
Each feature follows same pattern:
  - screens/: Full-page screens
  - components/: Feature-specific reusable components
  - models/: TypeScript types + mock data
  - services/: API calls (TODO: replace mocks)

UI Library:
- primitives/: Base components (GlassPanel, Button)
  * No business logic
  * Highly reusable
  * Accept style props
- components/: Composite components (Card, Pill)
  * Combine primitives
  * Minimal logic
  * Reusable across features

File Naming Conventions
=======================

Components:    PascalCase.tsx    (Button.tsx, GlassPanel.tsx)
Screens:       PascalCase.tsx    (TradeScreen.tsx)
Utilities:     camelCase.ts      (formatCurrency.ts)
Types/Models:  index.ts          (models/index.ts exports types)
Services:      PascalCase.ts     (AssistantService.ts)
Config:        lowercase/kebab   (app.json, babel.config.js)
